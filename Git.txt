# Git: Version control tool

# Github: A cloud for uploading project tracked using Git
  - Alternatives:
    - Gitlab
    - Bitbucket

# Terms:
  - repository: Project folder (Being tracked by Git)
    - local repository: Local project folder
    - remote repository: Project folder on Github
  - stage: Prepare files to be committed
  - commit: Save changes
  
# Local Workflow
  - Working directory: Make changes (Red)
  - Staging area: Changes ready to be 'commited' are moved here (Green)
  - Local repository (.git): Commited changes are saved here

# Steps to upload a project on Github
  - Github:
    - Create a new remote repository
      - name: Name of the project
      - description: Description of the project
      - visibility: Private/Public
        - Public (Open source)
      - README.md: Project description file
  - Local:
    - First time:
      - git init: Initialize a local git repository
      - git add: Stage files
        - git add file1.html: Stage file1.html
        - git add .: Stage all files
      - git commit -m "Message": Commit changes

    - Generic
      - git status: Current status of local repository
  - Working with Remotes


# Branches:
  - Default (master/main)
  - Separate branches can be created to work on individual featurs. For eg., in Amazon: Separate branches for Amazon Pay, Shopping, Prime Video features
